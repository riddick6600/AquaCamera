# Техническое задание: AquaCamera

## 1. Общая информация

**Название проекта:** AquaCamera  
**Тип приложения:** мобильное приложение на Flutter  
**Целевая платформа:** iOS  
**Путь проекта:** `/Users/gd/Projects/AquaCamera`  
**Стейт-менеджер:** BLoC  

## 2. Цель проекта

Создать Flutter-приложение для iOS, которое позволяет пользователю фотографировать через камеру устройства и автоматически сохранять полученные изображения во внутреннюю галерею приложения.

Приложение не должно использовать системную галерею iOS для просмотра, выбора или сохранения изображений.

## 3. Важные ограничения

Приложение должно работать только со своей локальной галереей.

Запрещено:

- открывать стандартную галерею iOS для выбора изображений;
- брать изображения из системной фотоплёнки;
- сохранять изображения в системную фотоплёнку;
- запрашивать разрешения на доступ к фото iOS, если они не нужны;
- использовать системный просмотрщик фотографий как основную галерею приложения.

Разрешено:

- использовать камеру устройства для создания новых фотографий;
- сохранять фотографии во внутреннее хранилище приложения;
- отображать сохранённые фотографии внутри собственного UI приложения;
- удалять фотографии только из локального хранилища приложения.

## 4. Основной пользовательский сценарий

1. Пользователь открывает приложение.
2. Приложение отображает собственную галерею сохранённых фотографий.
3. Пользователь нажимает кнопку камеры.
4. Приложение запрашивает разрешение на использование камеры, если оно ещё не выдано.
5. Пользователь делает фотографию.
6. Фотография сразу сохраняется во внутреннюю память приложения.
7. Сохранённая фотография появляется в галерее приложения.
8. После перезапуска приложения фотографии должны оставаться доступными.

## 5. Функциональные требования

### 5.1 Камера

Приложение должно:

- открывать камеру iPhone;
- позволять сделать фотографию;
- получать файл сделанной фотографии;
- сохранять фотографию во внутреннюю директорию приложения;
- добавлять запись о фотографии в локальный список медиа;
- отображать новое изображение в галерее приложения без ручного обновления экрана.

### 5.2 Галерея приложения

Галерея должна быть реализована внутри приложения.

Галерея должна:

- показывать только изображения, созданные через это приложение;
- загружать изображения из локального хранилища приложения;
- отображать изображения в виде сетки;
- поддерживать состояние пустого списка;
- поддерживать просмотр изображения в полноэкранном режиме;
- не обращаться к системной галерее iOS.

### 5.3 Локальное хранение

Фотографии должны сохраняться в директорию приложения, например:

```text
ApplicationDocumentsDirectory/aqua_camera/images/
```

Для каждой фотографии необходимо хранить метаданные:

- уникальный идентификатор;
- локальный путь к файлу;
- дата и время создания;
- размер файла;
- ширина и высота изображения, если эти данные доступны без существенного усложнения;
- источник изображения: `camera`.

Метаданные можно хранить в локальной базе или локальном JSON-хранилище. Конкретный способ хранения выбирается при реализации, но данные должны переживать перезапуск приложения.

### 5.4 Удаление фотографий

Приложение должно позволять удалить фотографию из галереи приложения.

При удалении необходимо:

- удалить файл изображения из локальной директории приложения;
- удалить запись из локальных метаданных;
- обновить состояние галереи.

### 5.5 Ошибки и разрешения

Приложение должно корректно обрабатывать:

- отказ в доступе к камере;
- отмену фотографирования пользователем;
- ошибку сохранения файла;
- ошибку чтения локального хранилища;
- отсутствие сохранённых фотографий.

При ошибке пользователь должен видеть понятное сообщение внутри приложения.

## 6. iOS-требования

Приложение должно запускаться и работать на реальном iPhone.

В `ios/Runner/Info.plist` необходимо добавить:

```xml
<key>NSCameraUsageDescription</key>
<string>Приложению нужен доступ к камере для создания фотографий.</string>
```

Разрешения для системной фотогалереи добавлять не нужно, так как приложение не должно читать или записывать изображения в фотоплёнку iOS.

Не добавлять без необходимости:

- `NSPhotoLibraryUsageDescription`;
- `NSPhotoLibraryAddUsageDescription`.

## 8. BLoC

Состояние экрана должно управляться через BLoC.

Основные события:

- `LoadLocalPhotosRequested`;
- `TakePhotoRequested`;
- `PhotoCaptured`;
- `DeletePhotoRequested`;
- `OpenPhotoRequested`, если полноэкранный просмотр управляется через BLoC;
- `RetryRequested`.

Основные состояния:

- `CameraGalleryInitial`;
- `CameraGalleryLoading`;
- `CameraGalleryLoaded`;
- `CameraGalleryEmpty`;
- `CameraGalleryFailure`;
- `CameraPermissionDenied`.

BLoC не должен напрямую работать с файловой системой, камерой или платформенными API. Эти задачи должны быть вынесены в репозитории и сервисы.

## 9. UI-требования

Главный экран должен содержать:

- верхнюю панель с названием приложения;
- кнопку запуска камеры;
- сетку локальных фотографий;
- состояние пустой галереи;
- индикатор загрузки;
- сообщение об ошибке;
- возможность открыть фотографию на весь экран;
- возможность удалить фотографию.

Интерфейс должен быть адаптирован под iPhone и корректно работать в портретной ориентации.

## 13. Что не входит в первую версию

- выбор изображений из системной галереи iOS;

- синхронизация с облаком;
- авторизация пользователей;
- редактирование изображений;
- видео;
- Android-версия.
